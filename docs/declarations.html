Declarations Language

The Declarations textarea on the setup screen accepts three kinds of declarations: type declarations, function definitions, and theorem declarations. You can include any number of each, in any order.

Built-in Type: Int

Int is a built-in type representing integers. You do not need to declare it. Lowercase variables (like x, y, n) are automatically available as integer variables when no declarations are present.

Type Declarations

A type declaration defines an inductively defined data type with one or more constructors.

Syntax

type TypeName
  | CtorName : TypeName
  | CtorName : (ParamType1, ParamType2) -> TypeName

Rules:

  • Type names must start with an uppercase letter (e.g., List, Tree).
  • Constructor names also start with a lowercase letter (e.g., nil, cons, leaf, node).
  • Each constructor line starts with |.
  • A constructor with no parameters just has a return type: nil : List.
  • A constructor with parameters uses arrow syntax: cons : (Int, List) -> List.
  • Parameter types can be Int or any declared type name.
  • All constructors of a type must return that type.

Examples

A linked list of integers

type List
  | nil : List
  | cons : (Int, List) -> List

This defines a List type with two constructors: nil (the empty list) and cons (which takes an integer and a list to produce a longer list).

A binary tree

type Tree
  | leaf : Tree
  | node : (Tree, Tree) -> Tree

Function Definitions

A function definition declares a recursive function by pattern matching on its arguments.

Syntax

def funcName : (ParamType1, ParamType2) -> ReturnType
  | funcName(pattern1) => body1
  | funcName(pattern2) => body2

Rules:

  • The first line is the type signature: parameter types and return type.
  • Each case starts with | and shows the function applied to a pattern.
  • Patterns can be:
    • Variables: a, L, n — match anything
    • Constructors: nil, cons(a, L), node(L, R) — destructure values
  • The body after => is an arithmetic expression that can use:
    • Variables from the pattern
    • Recursive calls to any defined function
    • Constructors as function calls (e.g., cons(a, L))
    • Arithmetic: +, -, *, ^

Examples

List length

def len : (List) -> Int
  | len(nil) => 0
  | len(cons(a, L)) => 1 + len(L)

Tree size (number of nodes)

def size : (Tree) -> Int
  | size(leaf) => 1
  | size(node(L, R)) => 1 + size(L) + size(R)

Filter positive elements

def positives : (List) -> List
  | positives(nil) => nil
  | positives(cons(a, L)) => if a < 0 then positives(L) else cons(a, positives(L))

Functions with If-Else (Conditional Definitions)

A case body can use an if ... then ... else ... expression. The condition must be an inequality using < or <=. 

| funcName(pattern) => if condition then expr1 else expr2

When a case uses if-else, it generates two definitions instead of one. For example, if positives has case 2 with an if:

  • positives_2a: positives(cons(a, L)) = positives(L) — when a < 0
  • positives_2b: positives(cons(a, L)) = cons(a, positives(L)) — when 0 <= a

See the Rules Reference for how to use these conditional definitions in proofs.

Definition Naming

Each function case is automatically assigned a name used in proof rules:

CaseNameExample
Simple case i funcName_i len_1, len_2
If-else case i, then branch funcName_ia positives_2a
If-else case i, else branch funcName_ib positives_2b

These names are used with the defof and undef rules. See Rules Reference.

Theorem Declarations

A theorem declaration states a named result that you have already proved (or that you intend to use as a lemma). Declared theorems can be applied in later proofs using the apply rule.

Syntax

theorem thmName (param1, param2 : Type)
  | conclusion

Or, for a theorem with a premise (an implication):

theorem thmName (param1, param2 : Type)
  | premise => conclusion

Rules:

  • Theorem names start with a lowercase letter (e.g., len_concat).
  • Parameters are grouped by type: (x, y : Int) or (S, T : List).
  • The body after | is a formula (equation or inequality).
  • If the theorem has a premise, write | premise => conclusion. When using it in a proof, you must cite a known fact that establishes the premise.

Examples

A theorem about list length and concatenation

theorem len_concat (S, T : List)
  | len(concat(S, T)) = len(S) + len(T)

This declares that len_concat is a known result. In another proof, you can use apply len_concat to replace len(concat(A, B)) with len(A) + len(B).

Theorem declarations in the Declarations area are lemmas you have already proved separately. The tool treats them as axioms when checking proofs that cite them. Prove them first by using each theorem as the goal in its own proof session.

Variables

Variables are not declared in the Declarations area. Instead, they are declared as parameters of a theorem in the Prove field: 

theorem thmName (xs : List) (n : Int)
  | formula

Lowercase variables default to Int. For variables of user-defined types (like List or Tree), you must declare the type explicitly in the theorem's parameter list.

To do induction on a variable, it must have an inductively defined type (not Int). Declare it as a theorem parameter with the appropriate type.

Putting It All Together

A complete declarations block:

type List
  | nil : List
  | cons : (Int, List) -> List

def len : (List) -> Int
  | len(nil) => 0
  | len(cons(a, L)) => 1 + len(L)

def echo : (List) -> List
  | echo(nil) => nil
  | echo(cons(a, L)) => cons(a, cons(a, echo(L)))

theorem len_echo (xs : List)
  | len(echo(xs)) = 2 * len(xs)

This declares a List type, two functions (len and echo), and a previously proved theorem len_echo. Variables like xs are declared in the theorem statement itself, not in the declarations area.
