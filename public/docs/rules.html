Rules Reference

This page documents every rule you can type into the forward (top) and backward (bottom) input fields of a calculation proof. All rules support Tab-completion.

Known Facts

Many rules can cite known facts by number. Known facts include:

  1. Givens you entered on the setup screen (numbered starting from 1).
  2. Induction hypotheses generated by induction cases (numbered after the givens).
  3. Case conditions from proof by cases (numbered after previous facts).

Fact numbers are shown in the proof view next to each given, induction hypothesis, or case condition.

Forward Rules (Top Input)

Forward rules transform the current expression (the bottom of the forward chain) into a new expression.

Algebra: = expr, < expr, <= expr

Assert that the current expression equals (or is less than / less-or-equal to) the given expression. Comfy uses a built-in decision procedure to verify this automatically over integer arithmetic. 

-- Assert current expression equals 2*x + 3
= 2*x + 3

-- Assert with known fact references
= 1 + len(L) 1

-- Inequality step
< sum(positives(L)) 2

The fact references (numbers after the expression) tell the decision procedure to use those known equations/inequalities when checking validity. For example, = 1 + len(L) 1 means "the current expression equals 1 + len(L), using fact 1."

The algebra decision procedure handles polynomial arithmetic over integers. It can verify identities like (a+b)^2 = a^2 + 2*a*b + b^2 automatically. For steps involving uninterpreted functions (like len), you typically need to cite a known fact.

Substitution: subst N

Substitute using known fact N. If fact N is an equation A = B, this replaces occurrences of A with B in the current expression. 

-- Given: 1. len(L) = 0
-- Current: 1 + len(L)
subst 1
-- Result: 1 + 0

For inequalities: if fact N is A <= B, substituting A with B at a positive position gives <=; at a negative position (like inside a subtraction) the direction flips.

With explicit result: subst N (expr)

If the substitution is ambiguous (multiple occurrences) or for clarity, you can provide the expected result in parentheses: 

subst 1 (1 + 0)

Reverse Substitution: unsub N

The reverse of subst. If fact N is A = B, this replaces occurrences of B with A. 

-- Given: 1. len(L) = 0
-- Current: 1 + 0
unsub 1
-- Result: 1 + len(L)

Apply Definition: defof name

Apply a function definition by matching the definition's left-hand side (pattern) in the current expression and replacing it with the right-hand side (body). 

-- Definitions: len_1: len(nil) = 0
--              len_2: len(cons(a, L)) = 1 + len(L)
-- Current: len(nil)
defof len_1
-- Result: 0

-- Current: len(cons(x, xs))
defof len_2
-- Result: 1 + len(xs)

The definition name follows the pattern funcName_N where N is the case number (1-indexed). For conditional definitions, use funcName_Na (then branch) or funcName_Nb (else branch). See Definition Naming.

Conditional definitions: defof name N

For conditional definitions (from if-else function cases), you must cite the known fact that proves the condition holds: 

-- Definition: positives_2a: positives(cons(a, L)) = positives(L) if a < 0
-- Given: 2. a < 0
defof positives_2a 2
-- Uses fact 2 to verify the condition a < 0

With explicit result: defof name (expr)

When the definition could match multiple sub-expressions, provide the expected result: 

defof len_1 (0 + 0)

Reverse Definition: undef name

The reverse of defof. Matches the definition's right-hand side (body) in the current expression and replaces it with the left-hand side (pattern). 

-- Definition: len_1: len(nil) = 0
-- Current: 5 + 0
undef len_1
-- Result: 5 + len(nil)

This is commonly used in backward chains to "fold" an expression back into a function call.

Backward Tactics (Bottom Input)

Backward tactics work from the goal side upward. They determine a premise that, if proven, would establish the goal.

Algebra: expr =, expr <, expr <=

Assert that the given expression equals (or has the given relationship to) the current goal expression. Note the operator comes after the expression, opposite of forward rules. 

-- Current goal expression: (x + y)^2
(x^2 + 2*x*y + y^2) =
-- New goal expression: x^2 + 2*x*y + y^2

Parentheses around the expression are needed when it contains operators, to separate it from the = at the end. 

-- With fact references
(1 + len(L)) = 1 2

Substitution/Definition Tactics

subst, unsub, defof, and undef use the same syntax as forward rules but operate in reverse:

TacticEffect on goal
subst N Reverse a forward subst: replaces B with A in the goal (if fact N is A = B)
unsub N Reverse a forward unsub: replaces A with B in the goal
defof name Reverse a forward defof: replaces body with pattern in the goal
undef name Reverse a forward undef: replaces pattern with body in the goal
A useful mental model: backward undef len_1 in the bottom input has the same effect as forward defof len_1 would on the same expression — it unfolds the definition. The difference is just which chain it operates on.

Expression Syntax in Rules

Expressions within rules use standard arithmetic syntax:

xVariable—
f(x, y)Function/constructor callhighest
x^2Exponentiation (integer exponents only)high

-xUnary negationmedium
x + yAdditionlow
x - ySubtractionlow

In all cases, [refs] means optional space-separated fact numbers, and (expr) means an optional explicit result in parentheses can be appended to subst, unsub, defof, and undef.
