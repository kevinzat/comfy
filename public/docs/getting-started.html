Getting Started

When you open Comfy, you see the setup screen. This is where you describe the problem you want to prove. There are three sections: Declarations, Givens, and the goal (Prove).

1. Declarations (optional)

The Declarations textarea is where you define the types, functions, and variables used in your proof. This uses a small language described fully on the Declarations Language page.

If you are proving a fact about plain integers (like x^2 + 2*x*y + y^2 = (x + y)^2), you can leave this field empty — integer variables are available by default.

The placeholder text shows an example:

type List
  | nil : List
  | cons : (Int, List) -> List

def len : (List) -> Int
  | len(nil) => 0
  | len(cons(a, L)) => 1 + len(L)

var x : Int

The textarea highlights syntax errors as you type, with a red border and an error message showing where the problem was found.

2. Givens (optional)

Givens are formulas that you assume to be true. Click the + button to add a given, and type a formula (equation or inequality) into the input field. Click × to remove one.

Each given becomes a numbered fact that you can reference in your proof. For example, if you add two givens:

  1. x + y = 5
  2. x < 10

then you can cite fact 1 or fact 2 in your proof rules (e.g., = 5 1 to cite the first given).

The input field turns red if the formula has a syntax error.

3. Prove (required)

The Prove field is where you state the goal formula — the equation or inequality you want to prove. This is required.

Examples:

  • x^2 + 2*x*y + y^2 = (x + y)^2
  • len(echo(xs)) = 2 * len(xs)
  • sum(S) <= sum(positives(S))

Formulas

Formulas used in Givens and the Prove field have this syntax:

  • Operators: =, <, <=
  • Arithmetic: +, -, *, ^ (exponentiation)
  • Variables: lowercase letters like x, y, or names like xs
  • Function calls: len(xs), cons(a, L)
  • Constants: integer literals like 0, 1, 42
  • Parentheses for grouping: (x + y)^2

4. Check and Start

Once you have entered valid declarations (if any), givens, and a goal formula, click Check. This type-checks everything — it verifies that all referenced types exist, function bodies are well-typed, and the formulas use correctly typed expressions.

If the check succeeds, the button changes to Start. Click it to enter the proof view.

If type checking fails, an error message appears below the button explaining the problem (e.g., "unknown type" or "wrong number of arguments").

The Proof View

After clicking Start, you move to the proof view. This shows:

  • Your declarations displayed at the top (formatted with syntax highlighting)
  • Your givens as numbered facts
  • The goal you need to prove, with a method input below it

You then choose a proof method and build your proof step by step. There is a Show Text / Show HTML toggle at the bottom that switches between formatted display and plain text.

When the proof is complete, a Complete! message appears.
